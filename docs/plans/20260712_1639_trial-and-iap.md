# 21-Day Free Trial → One-Time $5.99 Unlock

**Status:** Implemented (code); pending App Store Connect + RevenueCat production setup · **Date:** 2026-07-12

## Context

Before shipping CaddyPoker to the App Store, the app is **free for 21 days**, after which the full game requires a **one-time $5.99 purchase** (no subscription). Apple provides no built-in free trial for one-time purchases (trials exist only for auto-renewing subscriptions), so the trial is **enforced in-app** and the unlock is a **non-consumable IAP**.

Decisions: app-enforced 21-day trial + non-consumable $5.99 unlock; trial start anchored in the **iOS Keychain** (survives reinstall); **RevenueCat** (`react-native-purchases`) for the purchase/entitlement layer. All code is **null-safe** — with no RevenueCat key set, the app runs in trial-only mode so nothing breaks before the store is configured.

## Design

### Access brain — `src/entitlements.ts`
A standalone Zustand store (`useEntitlements`) + module, kept **separate from `gameStore`** so its persist `version`/`partialize` are untouched.
- **Trial anchor:** stored in Keychain via `expo-secure-store` (`caddy.trialStartAt`). First launch writes `now` and fires `trial_started`. A monotonic `caddy.lastSeen` blunts clock-rollback trial extension.
- **Purchase:** RevenueCat entitlement `full_access`. `isUnlocked = customerInfo.entitlements.active['full_access'] != null`. Account-bound, so it survives reinstall via Restore.
- **Derived:** `trialDaysLeft()`, `hasAccess = isUnlocked || trialDaysLeft > 0`. Reactive hooks: `useHasAccess`, `useTrialDaysLeft`, `useIsUnlocked`, `useUnlockPrice`.
- **Actions:** `initEntitlements()` (boot), `purchaseUnlock()` (getOfferings → purchasePackage → refresh), `restore()`.
- **Analytics funnel** via existing `track()`: `trial_started`, `paywall_shown`, `purchase_started`, `purchase_completed`, `purchase_cancelled`, `purchase_failed`, `purchase_restored`. No PII.

### Boot + gating — `App.tsx`
- Boot calls `initEntitlements()` and blocks first paint on both persist hydration **and** entitlements ready (no paywall flash).
- **Resume guard:** if `!hasAccess` and `step` is a gameplay step (not home/menu/howToPlay/paywall), the router forces `paywall` — covers a backgrounded round resuming after expiry.

### Screens
- **New `src/screens/PaywallScreen.tsx`**: logo, perks, "Unlock — $5.99", Restore, and "Maybe later" **only while the trial is active** (hard gate once expired). Fires `paywall_shown`.
- **`HomeScreen`**: `start` routes to `paywall` when `!hasAccess`; a subtle "N days left in your free trial" badge shows during the trial.
- **`MenuScreen`**: the old "coming soon" Restore now calls `restore()`; an "Unlock Full Game" button (gold) routes to the paywall, hidden once `isUnlocked`.
- **`gameStore`**: `'paywall'` added to the `Step` union.

## Config
`app.json` → `extra`: `revenueCatKeyIos` (currently a RevenueCat **Test Store** key `test_…`; swap for the `appl_…` App Store key at launch), `unlockProductId = com.amontalbano.caddypoker.fullunlock`, `trialDays = 21`. Deps added: `react-native-purchases`, `expo-secure-store` (auto-added its config plugin).

## Remaining setup (outside the code)

**Apple / App Store Connect**
1. Sign the **Paid Applications Agreement** + banking/tax (Business → Agreements). IAP fails until active — do first.
2. Confirm the app record + In-App Purchase capability for `com.amontalbano.caddypoker`.
3. Create a **Non-Consumable** IAP: "Full Game Unlock", product id `com.amontalbano.caddypoker.fullunlock`, **$5.99**, display name/description + review screenshot. Submitted with the first app version. Optionally Family Sharing.
4. Create a **Sandbox tester**.
5. **Review notes:** explain the app-enforced 21-day trial and that the unlock is sandbox-testable.
6. **App Privacy:** declare purchase data as applicable (analytics already declared).

**RevenueCat**
1. Add the App Store app (bundle id) + App Store Connect IAP key.
2. Entitlement `full_access`; import product and attach.
3. Offering + package containing the product.
4. Copy the iOS **App Store public key** (`appl_…`) → `app.json` `extra.revenueCatKeyIos`.

## Verification
- `npx tsc --noEmit`; `npx expo run:ios --device "…" --configuration Release` (native rebuild for new pods).
- **Trial expiry:** seed `trialStartAt` 21+ days in the past (or lower `trialDays`) → New Round routes to paywall; badge shows "ended".
- **Purchase (sandbox / Test Store):** tap Unlock → complete → `isUnlocked` true, paywall dismisses, New Round works, badge gone; PostHog shows the funnel.
- **Restore across reinstall:** delete & reinstall → Keychain `trialStartAt` persists (trial not reset) → Restore returns the entitlement.
- **Offline:** cached entitlement + local trial grant access in airplane mode; a fresh purchase requires network.

## Notes
- Clock tampering can extend the trial (mitigated by the `lastSeen` guard + Keychain); acceptable for a $5.99 app.
- Expo Go can't run RevenueCat — test via device builds (the existing workflow).
- If the Paid Apps Agreement isn't active, `getOfferings()` is empty and purchases fail — the most common first-run snag.
